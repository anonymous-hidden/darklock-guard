Verification + Welcome Flow

DarkLock supports a clean onboarding experience where Welcome and Verification can be used together or separately. Both toggles can be ON at the same time; when both are ON, the welcome is sent after successful verification.

  • Welcome ON, Verification OFF: Sends welcome immediately on join.
  • Welcome OFF, Verification ON: Routes the user to verification; no welcome is sent.
  • Welcome ON, Verification ON: Verification runs on join; welcome is sent after approval (deferred).
  • Both OFF: No welcome or verification; normal entry.

Goodbye messages can also be enabled. Free servers use the default template; Pro servers can set a custom goodbye in the customization tab.

The system automatically removes users who fail to verify within the timeout period:

  • Checks every 5 minutes for expired verifications
  • Sends DM notification before kick (if enabled)
  • Creates audit log entry with reason
  • Dashboard event for staff visibility

Anti-Raid Protection

How Anti-Raid Works

DarkLock monitors several factors to detect potential raids:

  • Join Rate: Unusual number of joins in short time
  • Account Age: Multiple new accounts joining
  • Similar Usernames: Pattern-based usernames
  • No Avatar: Default Discord avatars

Configuration Options

Setting Default Description
Join Rate Threshold 10 users/minute Triggers raid detection
Account Age Minimum 7 days Minimum account age allowed
Lockdown Duration 30 minutes How long server stays locked
Verification Level Medium Required verification during raids

Manual Commands

/lockdown [duration] [reason]
/unlock [reason]
/raid-mode [on/off]
