üõ°Ô∏è 3. Security Features

Anti-Raid Protection

Automatically detects and prevents coordinated raid attacks by monitoring join patterns.

How It Works

  • Monitors member joins per minute
  • Triggers when threshold exceeded (default: 10 joins/min)
  • Automatically locks server and notifies staff
  • Logs all suspicious joins with timestamps

Configuration

/security setup
Set anti-raid threshold and actions

Anti-Spam Protection

Prevents message spam, duplicate content, and mass mentions.

  • Duplicate Detection: Max 5 identical messages in 10 seconds
  • Mention Limits: Max 5 mentions per message
  • Rapid Fire: Detects unusually fast message sending
  • Auto-Moderation: Deletes spam and issues warnings

Anti-Phishing Protection

Scans messages for malicious links and phishing attempts.

  • Checks against known phishing domain databases
  • Analyzes suspicious link patterns
  • Auto-deletes dangerous links
  • Warns users and logs incidents
  • Optional integration with VirusTotal, URLVoid, Google Safe Browsing

Anti-Nuke Protection NEW

Prevents compromised accounts from destroying your server.

Protected Actions

  • Channel Deletion: Max 5 channels deleted per minute
  • Role Deletion: Max 5 roles deleted per minute
  • Mass Bans: Max 10 bans per minute
  • Permission Changes: Monitors bulk permission modifications

Commands

/antinuke enable
Enable anti-nuke protection
/antinuke whitelist @user
Add trusted user to whitelist (bypasses limits)
/antinuke settings
Configure thresholds (role_delete_limit, channel_delete_limit, ban_limit)
‚ö†Ô∏è Important: Anti-nuke protection may interfere with legitimate bulk operations. Always whitelist administrators performing server maintenance.

üîí 5. Lockdown System NEW ADMIN

Overview

The professional lockdown system provides emergency server protection with multiple modes, automatic permission backup, and restoration capabilities.

Lockdown Modes

Full Lockdown

Locks all channels server-wide (default mode)

  • Removes SEND_MESSAGES permission from @everyone
  • Only staff can send messages
  • Permissions automatically backed up
  • Use for: Severe raids, widespread disruption

Soft Lockdown

Allows members to read but not send messages

  • Keeps VIEW_CHANNEL enabled
  • Removes SEND_MESSAGES only
  • Members can still see announcements
  • Use for: Temporary announcements, emergency broadcasts

Channels Lockdown

Lock specific channels only

  • Target specific channels by ID or mention
  • Leaves rest of server functional
  • Useful for isolating problematic areas

Commands

/lockdown mode:full [reason] [duration]
Full server lockdown - all channels locked
Duration: Optional auto-unlock timer (e.g., "1h", "30m")
/lockdown mode:soft [reason]
Soft lockdown - read-only mode
/lockdown mode:channels [channel1] [channel2] [reason]
Lock specific channels
Specify up to 25 channels
/unlockdown [reason]
End lockdown and restore all permissions

Additional Options

  • timeout_new_accounts: Auto-timeout members who join during lockdown (prevents raid continuation)
  • disable_invites: Pause all server invites during lockdown
  • disable_tickets: Prevent new ticket creation during lockdown
  • slowmode: Apply slowmode to all channels (0-21600 seconds)
  • notify_channel: Send lockdown announcements to specific channel

Example Usage

/lockdown mode:full reason:"Raid detected" duration:1h timeout_new_accounts:true disable_invites:true
Full lockdown for 1 hour with new member timeouts and invite pause

Automatic Features

  • Permission Backup: All channel permissions saved before lockdown
  • Automatic Restoration: Original permissions restored on unlock
  • New Member Handling: Members joining during lockdown auto-timed out
  • Dashboard Integration: View lockdown status and history in web dashboard
  • Staff Notifications: Admins notified via DM when lockdown triggered
  • Logging: All lockdown events logged with timestamps and reasons
‚ö†Ô∏è Emergency Use Only: Lockdown is a drastic measure. Use only during active attacks or emergencies. Always communicate with your community about why lockdown was triggered.

Dashboard Control

Lockdown can also be triggered from the web dashboard:

  • Click "Server Lockdown" in Quick Actions sidebar
  • Select mode and options
  • View active lockdown status in Security tab
  • Review lockdown history and logs
üí° Pro Tip: Test lockdown in a staging channel first. Use /lock on a single channel to verify permissions work correctly before using full server lockdown.

Overview

The verification system prevents bot accounts and raiders from accessing your server by requiring new members to complete a verification challenge.

Verification Methods

Button Verification

Simple click-to-verify system

  • Members click a button to verify
  • Instant role assignment
  • Best for: General communities, low-risk servers

Captcha Verification

Advanced image-based challenge

  • Members solve captcha image
  • Prevents automated bot accounts
  • Best for: High-security servers, during raids

Setup Commands

/verification enable / disable
Toggle verification system on/off

How It Works

  1. New member joins server
  2. Bot sends verification prompt in verification channel
  3. Member completes verification (button click or captcha)
  4. Bot assigns verified role automatically
  5. Member gains access to server channels

Configuration Tips

  • Set verification channel permissions so only unverified can see it
  • Configure verified role to access all member channels
  • Use captcha mode during active raids for better protection
  • Monitor verification queue in dashboard for stuck verifications

Scaling Recommendations

For Small Servers (< 1,000 members)

  • Default settings work great
  • No additional configuration needed
  • 1GB RAM sufficient

For Medium Servers (1,000 - 10,000 members)

  • Enable database query caching
  • Increase message cache to 2000
  • 2GB RAM recommended
  • Consider dedicated hosting

For Large Servers (10,000+ members)

  • Use PostgreSQL instead of SQLite
  • Enable Redis for caching
  • 4GB+ RAM required
  • Dedicated server with SSD storage

üîê 17. Security Best Practices

Bot Token Security

‚ö†Ô∏è Critical: NEVER share your bot token. It's like a password for your bot. If leaked, regenerate immediately.

Token Best Practices

  • Store token in .env file, never in code
  • Add .env to .gitignore (never commit to GitHub)
  • Use environment variables in production
  • Rotate token every 90 days
  • Use different tokens for development/production

Permission Security

Role Hierarchy

  • Bot role must be above roles it moderates
  • Never give members Administrator unless necessary
  • Use specific permissions instead of Admin
  • Separate mod and admin roles with different permissions

Command Permissions

  • Use /permissions setup to restrict dangerous commands
  • Limit lockdown/backup commands to admins only
  • Restrict moderation commands to mod role
  • Review permissions quarterly

Server Security

Verification Settings

  • Enable verification for all new members
  • Use captcha during suspicious activity
  • Set verification timeout (5-10 minutes)
  • Remove unverified members automatically

Channel Security

  • Restrict permissions on important channels
  • Use slowmode in busy channels
  • Limit attachment permissions to trusted members
  • Disable @everyone mentions in public channels

Dashboard Security

Authentication

  • OAuth2 Discord login only (no custom passwords)
  • Sessions expire after 7 days
  • IP-based rate limiting prevents brute force
  • 2FA recommended for admin Discord accounts

API Key Security

  • Generate unique keys for each integration
  • Set expiry dates on API keys
  • Revoke unused keys immediately
  • Monitor API usage for anomalies
  • Use HTTPS only for API requests

Data Security

Database

  • Regular automated backups (daily)
  • Encrypt backups before storage
  • Store backups in multiple locations
  • Test restore process monthly

User Data

  • Only collect necessary user data
  • Automatic cleanup of old data
  • Respect user privacy settings

Hosting Options

Option 1: Local Hosting (Free)

Pros: Free, full control, no resource limits

Cons: Requires 24/7 PC, no redundancy, your IP exposed

Best for: Testing, development, small personal servers

Option 2: VPS (Virtual Private Server)

Pros: Full control, affordable ($5-10/month), always online

Cons: Requires Linux knowledge, manual setup

Best for: Medium to large servers, custom configurations

Providers: DigitalOcean, Linode, Vultr, OVH

Option 3: Platform-as-a-Service (PaaS)

Pros: Easy deployment, auto-scaling, free tiers available

Cons: Limited configuration, can be expensive at scale

Best for: Quick deployment, beginners, MVP testing

Providers: Heroku, Render, Railway, Fly.io

Option 4: Dedicated Server

Pros: Maximum performance, dedicated resources, high uptime

Cons: Expensive ($50-200/month), overkill for most

Best for: Very large servers (50k+ members), multiple bots

üîß 19. Troubleshooting & Common Issues

Bot Not Starting

‚ùå Error: "Invalid Token"
  • Check .env file has correct BOT_TOKEN
  • No spaces or quotes around token
  • Regenerate token in Discord Developer Portal if necessary
  • Restart bot after changing .env
‚ùå Error: "Cannot find module"
  • Run npm install to install dependencies
  • Delete node_modules and package-lock.json
  • Run npm install again
  • Check Node.js version is 18 or higher

Commands Not Working

‚ùå Slash Commands Not Showing
  • Wait 5 minutes for Discord to sync commands
  • Restart Discord app completely
  • Check bot has applications.commands scope in invite
  • Re-invite bot with proper permissions
‚ùå "Missing Permissions" Error
  • Ensure bot role is above target role
  • Grant Administrator permission (easiest)
  • Or grant specific permissions: Manage Server, Manage Roles, Manage Channels, Ban Members, etc.
  • Use /permissions check to diagnose

Dashboard Issues

‚ùå Cannot Access Dashboard
  • Check dashboard is running: pm2 status
  • Verify PORT environment variable
  • Check firewall allows port 3000 (or your PORT)
  • Ensure REDIRECT_URI matches exactly in Discord Developer Portal
‚ùå "Invalid OAuth2 Redirect"
  • Go to Discord Developer Portal
  • OAuth2 ‚Üí Redirects
  • Add: https://yourdomain.com/auth/callback
  • Ensure REDIRECT_URI in .env matches exactly
‚ùå Dashboard Not Updating
  • Check WebSocket connection in browser console
  • Click "Refresh Data" button
  • Clear browser cache and cookies
  • Check bot is online in Discord
  • Restart dashboard: pm2 restart all

Security Feature Issues

‚ùå Verification Not Working
  • Run /verification setup again
  • Ensure verification channel exists
  • Check verified role exists and bot can assign it
  • Verify bot role is above verified role
  • Check verification queue in dashboard
‚ùå Anti-Raid Not Triggering
  • Check anti-raid is enabled in dashboard
  • Verify threshold settings (default: 10 joins/min)
  • Ensure security log channel is configured
  • Check bot has lockdown permissions
‚ùå Lockdown Not Working
  • Bot needs Manage Channels permission
  • Bot role must be above @everyone
  • Check for channel permission overrides
  • Use /lockdown mode:channels for specific channels first to test

Database Issues

‚ùå "Database is locked"
  • Another process is using the database
  • Check for multiple bot instances running
  • Run pm2 delete all && pm2 start src/bot.js
  • If persistent, delete .db-wal and .db-shm files
‚ùå Data Not Persisting
  • Check DATABASE_PATH in .env
  • Ensure data directory has write permissions
  • Verify disk space is available
  • Check database file exists and isn't corrupted

  • Medium servers (100-1k): Anti-raid: 10, Anti-spam: Moderate
  • Large servers (1k-10k): Anti-raid: 15, Anti-spam: Moderate
  • Very large (10k+): Anti-raid: 20, Anti-spam: Lenient

Common Terms

A-D

  • Anti-Nuke: Protection against malicious mass actions (channel deletion, bans, etc.)
  • Anti-Raid: System that detects coordinated join attacks
  • API Key: Authentication token for API access
  • Autorole: Role automatically given to new members
  • Case: Moderation action record with ID, reason, and timestamp
  • Dashboard: Web-based control panel for bot management

E-L

  • Embed: Formatted message with colors, fields, and images
  • Guild: Discord's term for a server
  • Lockdown: Emergency mode that restricts server access
  • Leveling: XP-based ranking system rewarding activity

M-R

  • Moderation: Actions taken to enforce rules (ban, kick, warn, etc.)
  • OAuth2: Authentication system for dashboard login
  • Permission Override: Channel-specific permission changes
  • Phishing: Malicious links attempting to steal credentials
  • Raid: Coordinated attack where many accounts join to spam/disrupt
  • Reaction Role: System where users get roles by clicking reactions

S-Z

  • Slash Command: Discord's new command format (/command)
  • Slowmode: Delay between messages users can send
  • Spam: Repeated unwanted messages
  • Ticket: Private support channel between user and staff
  • Timeout: Temporary mute preventing user from sending messages
  • Token: Bot's authentication credential (keep secret!)
  • Toxicity: Harmful behavior like harassment, hate speech
  • Verification: Process new members must complete to access server
  • WebSocket: Real-time communication protocol for instant updates
  • Whitelist: List of trusted users exempted from restrictions

Technical Terms

  • API: Application Programming Interface - way to interact with bot programmatically
  • CDN: Content Delivery Network - fast file hosting
  • CRUD: Create, Read, Update, Delete - basic database operations
  • Environment Variable: Configuration value stored outside code (.env file)
  • JSON: JavaScript Object Notation - data format
  • Latency: Delay between command and response (measured in ms)
  • Migration: Database schema update
  • REST: Architecture for APIs using HTTP requests
  • SQLite: Lightweight file-based database
  • VPS: Virtual Private Server - remote computer for hosting
‚ùì 26. Frequently Asked Questions

General Questions

Is this bot free?

Yes! The bot is completely free and open-source. You only pay for hosting if you choose a paid hosting provider. Some optional features (AI) require third-party API keys which may have costs.

Do I need coding knowledge to use this bot?

No coding required for basic usage. The setup wizard and dashboard make configuration easy. You only need basic Discord server management knowledge.

Can I use this bot on multiple servers?

Yes! Invite the bot to as many servers as you want. Each server has independent settings and data.

Is my data safe?

Yes. All data stored locally in your database. Bot doesn't send data to external services (except for optional AI features). Regular backups recommended.

Setup & Configuration

What permissions does the bot need?

Minimum: Manage Server, Manage Roles, Manage Channels, Ban Members, Kick Members, Timeout Members, Manage Messages, View Channels, Send Messages. Administrator permission recommended for easiest setup.

How long does setup take?

5-10 minutes with the setup wizard. Additional features (verification, tickets, leveling) can be enabled later.

Can I customize the bot's responses?

Yes! Change language, create custom commands, modify embed colors, and customize welcome messages. Advanced users can edit locale files for complete customization.

How do I change the bot's prefix?

This bot uses slash commands (/command), which don't use prefixes. All commands work with /.

Security Features

Will anti-raid block legitimate member joins?

Rarely. Default threshold (10 joins/min) only triggers during actual raids. Legitimate growth patterns won't trigger it. Threshold can be adjusted for your server's size.

Does verification prevent all bot accounts?

No system is 100% effective, but verification stops 95%+ of bot accounts. Captcha verification is more effective than button verification.

Can I whitelist trusted members from security checks?

Yes. Anti-nuke supports whitelisting. Admins and bot role automatically exempted from most checks.

Moderation

Can users appeal bans?

Not built-in yet. Create a separate appeal server or use tickets in another server where banned users can submit appeals.

How long are moderation logs kept?

Forever by default. Old logs (90+ days) can be archived with /database cleanup to save space.

Can I see who performed each moderation action?

Yes. Every case includes moderator ID, timestamp, and reason. View with /cases recent or in dashboard.

Dashboard

Do I need to pay for the dashboard?

No. Dashboard included free. You only pay for hosting if using paid hosting service.

Who can access the dashboard?

Anyone with admin or moderator role in your Discord server. Permissions automatically synced.

Is the dashboard secure?

Yes. OAuth2 authentication, role-based access control, session management, and HTTPS encryption (if configured).

Technical Questions

What database does it use?

SQLite by default (no setup required). Can switch to PostgreSQL or MongoDB for very large servers (10k+ members).

Can it handle large servers?

Yes. Tested with servers up to 50k members. For 100k+, enable clustering. Most servers under 10k members work perfectly with default settings.

Does it support Discord's new features?

Yes. Uses Discord.js v14 with full support for slash commands, buttons, select menus, threads, forums, and all modern Discord features.

Can I contribute to development?

Absolutely! Bot is open-source. Fork on GitHub, make improvements, submit pull requests. See CONTRIBUTING.md for guidelines.
